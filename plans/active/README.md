# Active ExecPlans

Store only currently active ExecPlans for non-trivial and high-risk tasks in this directory.

## Rules
- Start from `docs/templates/execplan_template.md`.
- Follow trigger, quality, and lifecycle rules in `PLANS.md`.
- Keep one ExecPlan per active task.
- Use naming: `YYYY-MM-DD_<short-task-name>_execplan.md`.
- Move plans to `plans/completed/` once the task is `Complete`, `Partial`, `Cancelled`, or `Superseded`.
